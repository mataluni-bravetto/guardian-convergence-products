## The Pattern

1. **Intention** — What outcome you care about
2. **Constraint** — Stack, style, boundaries
3. **Recursion** — Model explores under constraints
4. **Validation** — Verify output meets requirements
5. **Correction** — Adjust and iterate
6. **Convergence** — Stop when it meets your bar

**Same loop for coding, docs, planning, communication.**

---

## How It Works

### 1. Intention
**What outcome do you actually care about?**

❌ "Add a feature"  
✅ "Add user authentication behind a feature flag"

**Be specific. Be clear. Be outcome-focused.**

---

### 2. Constraint
**Define your boundaries.**

- Stack: React + TypeScript
- Style: Functional components, hooks only
- Boundaries: Use existing API endpoints
- Non-negotiables: Must work with existing routing

**Constraints create clarity. Clarity creates convergence.**

---

### 3. Recursion
**Let the model explore under constraints.**

Provide intention + constraints + context.

**The model explores. You guide.**

---

### 4. Validation
**Verify output meets requirements.**

Use validation toolkit or manual review.

**Validation catches failures before production.**

---

### 5. Correction
**Adjust and iterate.**

If validation fails, clarify intention or tighten constraints.

**Correction is part of the loop, not failure.**

---

### 6. Convergence
**Stop when it meets your bar.**

✅ Code works  
✅ Meets constraints  
✅ Passes validation  
✅ Feels right

**Convergence is when you stop iterating.**

---

## Example

**Task:** "Add user authentication"

**Intention:** "Add user authentication behind a feature flag, using existing API endpoints"

**Constraints:** React + TypeScript, functional components, use existing `/api/auth` endpoints

**Recursion:** Ask AI to generate components under these constraints

**Validation:** Run validation toolkit, check for phantom APIs, verify security

**Correction:** Fix any issues found

**Convergence:** ✅ Code works, passes validation, meets all constraints
